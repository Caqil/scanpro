# VPS Requirements for ScanPro KubeSphere Deployment

## Minimum Cluster Configuration
For a production-grade ScanPro deployment with KubeSphere, the following infrastructure is recommended:

### Master Node (1 required)
- **CPU**: 4 cores minimum, 8 cores recommended
- **RAM**: 8GB minimum, 16GB recommended
- **Storage**: 50GB minimum SSD storage
- **OS**: Ubuntu 20.04 LTS
- **Network**: Static IP and public internet access

### Worker Nodes (minimum 2 recommended)
- **CPU**: 8 cores minimum, 16 cores recommended
- **RAM**: 16GB minimum, 32GB recommended
- **Storage**: 100GB minimum SSD storage
- **OS**: Ubuntu 20.04 LTS
- **Network**: Static IP and public internet access

### Storage Node (optional)
- **CPU**: 4 cores minimum
- **RAM**: 8GB minimum
- **Storage**: 500GB minimum SSD storage
- **OS**: Ubuntu 20.04 LTS
- **Network**: Static IP and private network connection to cluster

## Network Requirements
- All nodes must be on the same network with unfiltered communication between them
- Ports required:
  - 22 (SSH)
  - 6443 (Kubernetes API)
  - 2379-2380 (etcd)
  - 10250-10252 (Kubelet, Kube-scheduler, Kube-controller)
  - 30000-32767 (NodePort Services)
  - 80, 443 (HTTP/HTTPS)
  - 179 (Calico BGP if used)
  - 8080 (KubeSphere console)

## Provider Recommendations
Any cloud provider or bare metal setup that meets the above requirements will work.
Recommended providers include:

- **Digital Ocean**: Droplets with dedicated CPUs
- **Linode**: Dedicated CPU instances
- **Hetzner**: Dedicated servers or cloud instances
- **AWS**: EC2 instances (t3a.xlarge or better)
- **GCP**: Compute Engine instances (e2-standard-8 or better)
- **Azure**: D-series VMs

## Scaling Considerations
For larger ScanPro deployments with high traffic:

- Scale worker nodes horizontally (add more nodes)
- Consider dedicated database nodes
- Set up separate storage backends
- Implement a CDN for static content
- Configure proper resource limits for all containers

## Special Notes
- Docker and Kubernetes versions must be compatible with KubeSphere
- NTP must be properly configured across all nodes
- Ensure all VPS instances are in the same region for optimal performance
